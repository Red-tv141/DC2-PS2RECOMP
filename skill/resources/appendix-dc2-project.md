## §3 Build & Smoke (VS generator + MSBuild — NOT the skill's default Ninja path)

DC2's `build64` is the **Visual Studio generator** (MSBuild), so the skill's generic
"`cmake --build build64`" advice is WRONG here — a bare build defaults to Debug `ALL_BUILD` =
huge rebuild. **Always `--config Release --target <…>`:**

```powershell
cmake --build D:\ps2r\dc2\build64 --config Release --target ps2_runtime -- /m:1
cmake --build D:\ps2r\dc2\build64 --config Release --target dc2_runner -- /m:1 /p:BuildProjectReferences=false
```
- Build via the **PowerShell tool**, NOT Bash `cmd /c` (the latter silently no-ops → stale exe).
- **Verify a change landed in the binary:** `grep -c <marker> build64/Release/dc2_runner.exe`.
- Benign link warnings only: `LNK4006` (getGameName) + `LNK4088` (/FORCE). Anything else = real.
- **`build_rt.bat` / `build_runner.bat` wrapper trap (bit a session hard):** they redirect all build
  output to `build_out.txt` / `runner_out.txt` respectively and end with `echo BUILD_EXIT:%ERRORLEVEL%`.
  So the batch's OWN exit code is the `echo`'s (**always 0**) — a PowerShell `$LASTEXITCODE` after
  calling the wrapper is MEANINGLESS. **Read the real result from the log file**: `Select-String
  'BUILD_EXIT|error C|error LNK|LNK1120'` on `build_out.txt` (runtime) / `runner_out.txt` (runner).
  A link failure (e.g. an unresolved symbol) reports `BUILD_EXIT:1` there AND **deletes the exe** —
  if you trusted the fake `EXIT=0` you'd run a stale/absent binary. `Test-Path …/dc2_runner.exe`
  after a link error.
- **Anonymous-namespace linkage trap:** most of `dc2_game_override.cpp` is inside `namespace {…}`.
  A `extern void foo();` declared *inside* that namespace binds to an anon (internal-linkage) symbol
  and fails to link against a global `foo()` in another TU. Declare cross-TU externs at **global
  scope, above** the anon namespace (the file already does this for `g_dc2TitleRockScope` etc.).

**Runner exe:** `dc2_runner.exe` (NOT `ps2EntryRunner.exe`). Active run command:
```
D:/ps2r/dc2/build64/Release/dc2_runner.exe D:/ps2r/dc2/SCUS_972.13
```

## §4 PCSX2 A/B — Raw TCP DebugServer

In addition to the `mcp__pcsx2__*` MCP tools (`12-pcsx2-mcp-playbook.md`), DC2 talks to the
DebugServer directly: **`127.0.0.1:21512`, newline-delimited JSON, field is `cmd` (NOT
`command`)**. Commands: `status` / `evaluate{expression}` /
`read_memory{address:int,size}` / `set_breakpoint{address}` / `resume` / `clear_breakpoints`;
poll `status.data.paused` after `resume`. Response `{ok,address,length,hex}`.
- **Hard constraint:** maps **EE RAM + scratchpad ONLY**. VU `0x1100xxxx` and GS `0x12000000`
  read **0**. For VU/GS internal state use an offline `.gs` dump (`ref/dumps/new_game.gs`).
- Reusable harnesses: `tools/run_f56_pcsx2_ab.ps1`, `tools/run_f55_pcsx2_ab.ps1`.
- **Convention:** leave PCSX2 paused with breakpoints cleared when done.
- **Raw-TCP quirks (G186):** `read_memory` caps at **256 bytes per call** (chunk your reads);
  the server is effectively **single-client** — a wedged/abandoned client connection blocks
  the port (CLOSE_WAIT zombies) until PCSX2 restarts; compact JSON from python
  (`json.dumps(..., separators=(",",":")) + "
"`) works fine.
- Extra MCP tools available beyond the playbook: `pcsx2_get_backtrace`,
  `ps2recomp_lookup_function`, `ps2recomp_list_overrides`.

**PCSX2 SOURCE as the semantic oracle (not just the runtime).** The PCSX2 source tree is at
`D:\ps2r\pcsx2-master`. For VU/COP2 codegen correctness read `pcsx2/VUops.cpp`, `pcsx2/VUflags.cpp`,
`pcsx2/VU.h` (lane order, FMAC flag rules, outer-product pairing, control-reg indices, `vuDouble`
clamp). This is how the COP2 lane-reversal + the F51.8 audit (6 defects) + the G71 VU1 MAC/STATUS
gaps were settled. Differential harness pattern: `PS2Recomp/ps2xTest/src/cop2_bug_characterization.cpp`
+ `tools/run_f51_8_cop2_characterization.ps1` (calls the live recompiler's emit fn, asserts the
emitted C++ vs the oracle — no game route needed). Known codegen bug + fix-up: `tools/fix_cop2_destmask.py`
(re-apply after every regen). Full write-up: `plans/bug-report-cop2-destmask-lane-reversal.md`,
`plans/phase-F51.8-fix-log.md`.

## §5 Headless Input Injection (testing infra)

No real Ghidra, no hands on the pad — tests drive input via env vars parsed in
`dc2_game_override.cpp`:
- **`DC2_PAD_INPUT="start..end:Alias[+Alias];…"`** — scripted 16-bit scePad button mask. scePad
  bits: Select 0x1, L3 0x2, R3 0x4, Start 0x8, Up 0x10, Right 0x20, Down 0x40, Left 0x80,
  L2 0x100, R2 0x200, L1 0x400, R1 0x800, Triangle 0x1000, Circle 0x2000, Cross 0x4000,
  Square 0x8000. **Scripted masks must be byte-swapped** (`swap16`) — the game reads pad bytes
  swapped; the recorder/replay handles this (`tools/replay_from_rec.py`).
- **`DC2_DUNGEON_PAD`** — same syntax, driven off the **host present loop** (use this in-dungeon;
  the `mgEndFrame`-hook clock dies in-dungeon because the dungeon calls `mgEndFrame` via direct
  `jal`, bypassing the hook → scripted input stops ~frame 697).
- **`DC2_RSTICK`** — scripted RIGHT analog stick (`RRight/RLeft/RUp/RDown`); `DC2_PAD_INPUT`
  leaves both sticks centred, so model-preview rotation needs this.
- Live host gamepad (XInput via raylib) also works (`DC2_NO_XINPUT=1` to disable;
  `DC2_KEYBOARD=1` for keyboard). In-dungeon free-roam is the **LEFT analog stick**, not D-pad.
- **The runner's menu rendering is too corrupted to ID screens visually** — distinguish screens
  by state (`LoopNo`, `titleInfo`, `DngStatus`, `DAT_01ecd62c`), not by frame captures.

## §7 Durable Rules Specific to DC2 Regen / Allocator

- **Live `recomp/` provenance:** the checked-in `recomp/` was generated from
  `config_auto_recomp_F56baseline.toml` (v9), NOT the root `config.toml` — the root file's
  stub/skip lists are INERT for the current build (e.g. `__ct__9CMapPartsFv` is in the root
  skip list yet has a full real body in `recomp/`). Check provenance before reasoning from a
  toml list.
- **Instrumentation placement:** diagnostics for generated functions go DIRECTLY into the
  `recomp/*.cpp` file (precedent: G19 in `recomp/Draw__8mgCFrameFPUi_0x137e10.cpp`; G186's
  temporary per-label sp checks). `PS2Recomp/ps2xRuntime/src/runner/*.cpp` belongs to the
  separate `ps2EntryRunner` target and does NOT shadow dc2_runner. Always keep a `.bak` and
  restore after diagnosis.
- **G186 (fixed, default-ON, kill `DC2_G186_NO_PREEMPTFIX=1`):** the `0xe3dc70` /
  "some levels cannot be loaded" crash class was F50.7's `Draw__4CMap@0x160b10` override
  `$ra=0` sentinel breaking under back-edge preemption → pc-zero recovery → `$sp` desync →
  stack-frame corruption (see `16-runtime-concurrency-threading.md §8`). Reusable G186
  diagnostics (all opt-in): `DC2_G186_BADPTR_DUMP=1` (32MB guest-RAM dump at first-bad-pc →
  `captures/g186_badpc_ram.bin`), `DC2_G186_WATCH=<hexaddr>` (dispatch-boundary word watch,
  ring dumped at crash), `DC2_G186_SPBAL=1` (sp-balance trampoline), `DC2_G186_SPTRACE=1`
  (EditLoop/EditDraw sp probes — their per-frame "desync" lines are a benign yield artifact,
  only in-body shifts matter).

- **Configs are inputs to a MANUAL `ps2_recomp` run**, not referenced by the runtime build.
  `ps2_recomp.exe`'s build cache hard-references the stale path `d:/ps2r/PS2Recomp` — fix +
  rebuild the recompiler before regenerating.
- **After any regen:** (1) re-apply `tools/fix_cop2_destmask.py`; (2) audit the **allocator
  family** for coherence — `malloc/_malloc_r/memalign/free/realloc/calloc` (+ `_r` variants)
  must ALL route to the runtime allocator OR all be recompiled; a half-runtime/half-recompiled
  allocator silently corrupts memory (texture/CLUT/menu corruption); (3) validate VU0-helper
  un-stub collisions (`Kernel/Stubs/VU.cpp` vs recompiled bodies; `/FORCE` picks one winner).
- **`registerFunction` overrides only fire for indirect `jr $t9`/`jalr`.** A direct `jal <addr>`
  is a direct C++ call → bypasses dispatch. To intercept direct callers, mark the target as a TOML
  stub and regenerate so calls dispatch dynamically, or override the caller itself.
- Override API is `PS2_REGISTER_GAME_OVERRIDE` + `runtime.registerFunction` /
  `ps2_game_overrides::bindAddressHandler` (see `04-runtime-syscalls-stubs.md` §5) — and a raw
  `bindAddressHandler` that doesn't advance `ctx->pc` infinite-loops; advance via
  `getRegU32(ctx,31)`.
